# Erlangルーターサービス

これは将来のクラスターレイのためのOTP側の設計と負荷テストエリアです。生産用WebTransportエッジは`relay-rs`のままであり、このサービスは大規模なルームのためのルーム/セッションのファンアウトセマンティクスを所有しています。

責任:

- ルーム/セッションの監視
- KGM1ストリームファンアウト
- 参加者の存在確認
- バックプレッシャー
- WebTransport/WebSocketゲートウェイの監視
- メトリクス

MVPはブラウザ内で完全に実行できます。このサービスはリモートコラボレーションと生産スケーリングのためのものです。

## 負荷テストハーネス

リポジトリのルートから実行します:

```sh
node services/erlang-router/load-test.mjs
```

コミットされたハーネスはDD-005トポロジーをモデル化しています: 3ノード、5,000サブスクライバー、1つのパブリッシャー、最新のみのローカルサブスクライバーメールボックス、ノード損失の隔離。p99リレーファンアウトレイテンシが>= 30 msの場合、または失敗したノードが非ローカルサブスクライバーに影響を与える場合は失敗します。

最新のローカル結果:

```json
{
  "nodes": 3,
  "subscribers": 5000,
  "frames": 180,
  "p99Ms": 1.07,
  "targetP99Ms": 30,
  "pass": true,
  "localOnlyDrop": true
}
```

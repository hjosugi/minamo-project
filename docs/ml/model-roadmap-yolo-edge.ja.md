# YOLO / Edge ML ロードマップ

## 1. カスタムモデルが必要な理由

MediaPipeは強力な汎用の顔/手/ポーズトラッキングを提供します。製品には以下のカスタムモデルが必要です：

- ドラムスティック
- ドラムキットのパーツ
- 手と物体の接触
- 動きのぼやけたスティック
- アバター特有の小道具
- 低照度での回復

## 2. モデルタスク

| タスク | モデルタイプ | 出力 |
|---|---|---|
| スティック検出 | 物体検出 / セグメンテーション | スティックボックス、マスク、先端 |
| ドラムパーツ検出 | 物体検出 | スネア、ハイハット、トム、シンバルゾーン |
| 手と物体の接触 | 分類器 | 接触確率 |
| 障害物回復 | 時系列モデル | 予測されたランドマーク |
| 品質分類器 | 軽量分類器 | 低照度、ぼやけ、悪いフレーミング |

## 2.1 MediaPipeタスク vs カスタムONNX

| エリア | MediaPipeタスク | カスタムONNXパス |
|---|---|---|
| 顔と手 | デフォルトパス; 強力なブラウザサポート | 専門モデルが現在の安定性を上回る場合のみ |
| スティック検出 | 専用モデルなし | ドラムスティックのための優先パス |
| 全身ポーズ | 良好なベースライン | KGM-023のためのオプションの高精度バックエンド |
| デプロイメント | シンプルなCDN/ローカル`.task`ファイル | モデルハッシュ、プロバイダー選択、量子化されたバリアントが必要 |

決定：MediaPipeをデフォルトのプライバシー保護トラッカーとして維持し、明示的なONNXモデル仕様の背後にYOLOスタイルのスティック検出器を評価します。既存のMediaPipeプラスヒューリスティックなドラムパスをスティック先端のp95エラー、ヒットリコール、誤ヒット率、p95レイテンシで上回るまでデフォルトで採用しないこと。コードレベルの決定記録は`createYoloStickDetectorBaselinePlan()`です。

## 3. デプロイメントターゲット

- ブラウザWebGPU経由のONNX Runtime Web
- ブラウザWASMフォールバック
- デスクトップRustサービス
- モバイルWebViewの将来
- ユーザーがオプトインした場合のみクラウドベンチマークランナー

ランタイム契約：

- `OnnxModelSpec`はURL、入力形状、出力、オプションのSHA-256、量子化、および優先プロバイダーを記録します。
- `detectMlRuntimeCapabilities()`はWebGPU、WebGL、WASM、WASMスレッド、SIMD、CPU、およびクロスオリジンアイソレーションサポートを記録します。
- `chooseExecutionProvider()`は最初にWebGPUを試し、必要に応じてWASM、次にCPUにフォールバックします。
- モデルの読み込みは、期待されるハッシュが存在する場合に`fetchAndVerifyModel()`または`verifyModelSpecBytes()`を呼び出さなければなりません。
- ONNX Runtime Webアダプターの境界は`OnnxRuntimeAdapter`です：初期化、オプションのウォームアップ、検出、およびランタイム依存関係をソルバーコードに漏らさずに廃棄します。

## 4. データセット計画

明示的な同意のもとでのみ収集します。

必要なラベル：

- スティックボックス
- スティック先端
- ドラムパーツポリゴン
- 手/指の可視/隠蔽
- ヒットタイムスタンプ
- ヒットゾーン
- オーディオオンセット

プライバシー保護のキャプチャ：

- デフォルトのエクスポートはランドマーク/ラベルのみで、4桁に丸められます。
- 生のフレームはオプトインであり、寄稿者が明示的にアップロードするまでローカルです。
- 各レコードにはラベルとデータセットライセンスが付随します。
- ドラム専用のビデオエクスポートには顔のぼかしが必要です。
- トラッカーUIは`minamo.dataset.tracker-sample.v1` NDJSONを`consent.rawMedia: false`でエクスポートします。
- ラベリングの詳細は`docs/ml/dataset-labeling-guide.md`にあります。

## 5. トレーニング戦略

- プロトタイプのために公開の物体検出モデルから始めます。
- スティックとドラム用に小さなモデルをファインチューニングします。
- エッジ用に量子化します。
- ONNXをエクスポートします。
- WebGPUとWASMでブラウザ内で検証します。

エクスポートマニフェスト：

```json
{
  "schema": "minamo.model-export.v1",
  "modelName": "stick-yolo-n",
  "format": "onnx",
  "quantization": "int8",
  "inputShape": [1, 3, 320, 320],
  "outputs": ["boxes", "scores"],
  "sha256": "<hex>"
}
```

ベンチマークハーネスの出力は`ModelBenchmarkResult`です：fps、平均レイテンシ、p95レイテンシ、バックエンド、およびオプションのメモリの高水準マーク。アダプターレベルのブラウザテストには`runModelBenchmark()`を使用し、WebGPUとWASMの行を同じレポートに保持します。

エクスポート計画は`createQuantizedModelExportPlan()`で表されます。カスタムモデルの最小出荷セットは：

- SHA-256を含む元のONNXマニフェスト
- fp16またはint8ブラウザバリアント
- WASMフォールバックパス
- マニフェスト内のライセンス文字列

## 6. 安全性とプライバシー

- 生のビデオはデフォルトでローカルに留まります。
- データセットのアップロードはオプトインです。
- ドラム専用トレーニングのための顔ぼかしオプション。
- 各データセットエントリにライセンスを付与します。

## 7. 研究ノート

YOLOスタイルのスティック検出は、スティック先端が小さく、速く、しばしば動きがぼやけるため、最初に評価する必要があります。完全なカスタムハンドパイプラインは、固定クリップセットでMediaPipeを上回るまで待つべきです。そうでなければ、ユーザーに見える利点がないのにメンテナンスが増えます。

デスクトップカメラの角度が悪い場合、電話のコンパニオンキャプチャは有用です。マルチカメラフュージョンは後で有用ですが、生のビデオフュージョンではなく、タイムスタンプ付きの独立したKGMストリームとして開始すべきです。

オプションのIMUスティックは高速ロールタイミングを改善できますが、ハードウェアコストとペアリングUXを導入します。IMUデータは補助的な信頼性の向上として扱い、必須ではありません。

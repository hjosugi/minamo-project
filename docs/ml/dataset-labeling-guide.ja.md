# スティックとドラムデータセットラベリングガイド

このガイドでは、トラッカーによって使用されるプライバシー保護データセットパスと、最初のYOLOスタイルのスティック検出器ベースラインについて説明します。

## キャプチャポリシー

- デフォルトのキャプチャは、ランドマーク、ラベル、品質スコア、キャリブレーションされたドラムゾーン、および手のターゲットのみです。
- 生のカメラフレーム、生の音声、キャンバススナップショット、サムネイル、およびメディアブロブは、デフォルトのNDJSONエクスポートには含まれません。
- 各レコードには、ラベル、ライセンス、タイムスタンプ、同意ブロック、および `rawMedia: false` が含まれます。
- ローカル実験のために生のビデオを収集する貢献者は、それをデフォルトのトラッカーエクスポートの外に保管し、ドラムのみのクリップを共有する前に顔をぼかさなければなりません。

## トラッカーのワークフロー

1. `tracker/` を開いてトラッキングを開始します。
2. スティック、接触、およびドラムラベルのために手を有効にします。
3. ヒットまたはゾーンラベルを収集する際に、Drummerセットアップを使用してゾーンをキャリブレーションします。
4. データセットキャプチャでラベルを選択します。
5. 単一の例のためにサンプルをキャプチャするには「Capture sample」を押すか、短いバーストのために2fpsでキャプチャを有効にします。
6. NDJSONをダウンロードし、ライセンスフィールドをファイルに添付したままにします。

## ラベル

| ラベル | 使用目的 | 必要なコンテキスト |
|---|---|---|
| `stick-tip` | スティックの先端ポイントまたは推測された先端軌道 | 手のターゲット、手首、ジェスチャー、品質 |
| `stick-box` | 検出器バウンディングボックスのトレーニングプラン | 手のターゲットとオプションのローカル生フレーム（デフォルトエクスポートの外） |
| `drum-hit` | ヒットのタイミングとゾーン分類 | キャリブレーションされたドラムゾーン、アクティブゾーンID |
| `hand-contact` | 手と物体の接触分類器 | 手のターゲット、カール、手首、自信 |
| `low-light` | 品質分類器のネガティブ例 | 品質スコアと警告状態 |
| `motion-blur` | ブレ検出器と高速スティックケース | 品質警告と手の速度 |
| `open-hand` | 手のベースラインと偽接触のネガティブ | 手のターゲット、ジェスチャー |
| `unlabeled` | 一時的なトリアージのみ | トレーニングの前に再ラベル付けが必要 |

## レコードスキーマ

トラッカーエクスポートは `minamo.dataset.tracker-sample.v1` NDJSON を使用します。各行には以下が含まれます：

- `label`、`license`、`createdAt`、`source`、および `capturedBy`
- `consent.rawMedia: false`
- 丸められた `face`、`pose`、`hands`、`quality`、および `drum` フィールド
- ミラー、手、ポーズ、ドラマー モード、解像度、およびfpsを再現するために必要なランタイム設定

このエクスポートは、ブラウザ側のモデル研究用に設計されており、YOLOまたは分類器トレーニングセットを構築するスクリプトによって消費できます。生フレームデータセットは、明示的なレビューを伴う別のオプトインパイプラインに存在する必要があります。

## ベースライン評価

最初のスティック検出器は、デフォルトで採用されるのではなく、以下の点で現在のMediaPipeプラスヒューリスティックパスを上回るまで評価されるべきです：

- スティック先端のp95エラー
- ヒットリコール
- 偽ヒット率
- WebGPUおよびWASMでのp95レイテンシ
- 低光量およびモーションブラーでの動作

ベースラインのコードレベルの決定記録として、`src/core/ml.ts` の `createYoloStickDetectorBaselinePlan()` を使用してください。

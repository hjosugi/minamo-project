# PR #220 検証証拠

このディレクトリは、PR #220 および追跡問題 #221 のための耐久性のある、編集された証拠フォーマットを定義します。これは、未実行のハードウェアチェックを合格に変えるものではありません。

## 再現可能なベースライン

Node.js 22 と `package.json` で固定されたパッケージマネージャを使用します：

```sh
pnpm --version
pnpm install --frozen-lockfile
sha256sum pnpm-lock.yaml
pnpm release:smoke
```

正確な検証コミット、ロックファイルハッシュ、環境、コマンド、および結果を記録します。汚れた作業ツリーは、差分が添付されておらず、結果が明示的に `BLOCKED` または `FAIL` とマークされていない限り、受け入れ可能な証拠ではありません。

## 証拠レイアウト

まず、追跡されたドキュメントの外でプライベート/生の証拠を収集します：

```text
evidence/pr-220/<UTC timestamp>/
  commit.txt
  pnpm-lock.sha256
  release-smoke.log
  ENVIRONMENT.md
  COMMANDS.md
  RESULT.md
  windows/
  webgpu/
  avatar-pack/
  iphone/
  multi-tracker/
  inochi2d/
  virtual-camera/
  drum/
  research-review/
```

`evidence/pr-220/` は、プライベートデバイスデータを含む可能性があるため、gitignoreされています。レビュー後、編集された再配布可能な記録のみを `docs/verification/pr-220/runs/<UTC timestamp>/` にコピーし、[INDEX.md](INDEX.md) からリンクします。

## 結果の意味

- `PASS`: 記録されたコミットで、すべての合格基準が実行され、合格しました。
- `FAIL`: 手順が実行され、少なくとも1つの必須基準が失敗しました。
- `BLOCKED`: 手順を完了できませんでした。所有者とブロック解除ステップをリストしてください。
- `N/A`: 基準が実際に適用されない場合。範囲を限定した理由と承認したレビュアーを示してください。ハードウェアの欠如や未完成のコードは `BLOCKED` であり、`N/A` ではありません。

[templates/](templates/) のステータス特有のファイルから始めます。必須フィールドを削除しないでください。記入された非製品サンプルは [sample/RESULT.md](sample/RESULT.md) にあります。

## プライバシーと編集

以下のものをコミットまたは添付しないでください：

- ルームトークン、API資格情報、クッキー、認証ヘッダー、または `.env` ファイル；
- プライベートCAキー、署名アイデンティティ、プロビジョニングプロファイル、または再利用可能な証明書；
- プライベート生カメラ/音声/ビデオ、バイオメトリック画像、またはライセンスのないアバター；
- 完全なローカルユーザー名、ホームパス、デバイスシリアル、LANアドレス、またはQRトークン。

生メディアは、記録されたすべての人が明示的に同意し、ライセンスが再配布を許可しない限り、ローカルに留まります。ハッシュ、派生モーションイベント、トリミングまたはぼかし処理されたキャプチャ、および編集されたログを優先してください。公開TLS証明書および証明書フィンガープリントは記録できますが、プライベートキーは記録できません。

公開前に、候補証拠をトークン、ローカルユーザー名、ホームディレクトリ、IPアドレス、およびプライベートキーのヘッダーについて検索します。`RESULT.md` に編集レビュアーと日付を記録します。

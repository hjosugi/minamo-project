## 完了した範囲

- 公式の Inox2D WebGL2 はリビジョン `df8413e6b0c525dbb880b4dca2bdf0a5d4b9aaba` に固定され、6つのメソッドアダプター契約の背後にラップされ、記録された SHA-256 値を持つローカル JS/WASM としてバンドルされています。
- `.inp` と `.inx` は拡張子ベースのファイル入力、ドラッグ＆ドロップ、および URL ロードパスを共有します。パーサーは、WASM 構築前にマジックとバウンド JSON ヘッダーを検証します。
- 名前付きパラメータメタデータは、ライブ/エクスポータブル `minamo.expression-map.v1` エディタに供給されます。頭のヨー/ピッチ/ロールはパペット範囲を使用し、まばたきと口のデフォルトは保守的な正規化名エイリアスを使用します。
- Inox2D は隠れたアルファ/ステンシル WebGL2 キャンバスに描画します。Three.js の `CanvasTexture` が既存の最終ビューワレンダラーを通じて合成します。
- アバターの置き換え、参加者の削除、初期化失敗、レンダリング失敗、およびページアンロードには、WASM オブジェクト、Three リソース、隠れた GL コンテキスト、およびキャンバスのための明示的な診断と廃棄パスがあります。

## 検証結果

- パーサー、マッピング、モックされた WASM ライフサイクル、廃棄の冪等性、および診断回帰テストは、すべての既存の JS/TypeScript テストで合格しました。
- Rust WASM フォーマットとクリッピーは警告を拒否して合格しました。
- プロダクションビルドは、ローカルの 593,966 バイトの WASM アセットを生成し、リリーススモークが合格しました。
- Chrome は、34 のパラメータを持つライセンスされた Aka 派生のリアルリグ/ペイロードを読み込み、例外なしで 7 秒間プロダクション WebGL2/WASM レンダーループを実行し、`Inochi2D 4/34 parameters` とステータス `open` を報告しました。

# ADR-002: アバターマッピングの前に安定性レイヤーが必要

## ステータス

承認済み

## コンテキスト

生のMLランドマークは揺れたり、反転したり、無効になったりすることがあります。アバターリグはこれらの失敗を視覚的に増幅します。

## 決定

すべてのML出力は、レンダリングまたは輸送の前に、品質ゲート、スムージング、解剖学的制約、およびリグクランプを通過しなければなりません。

## 結果

- わずかな追加レイテンシーは受け入れられます。
- 視覚的品質が向上します。
- すべての新しいアダプターは、信頼性と警告を提供しなければなりません。

## 検証

- 自動チェック: `pnpm test`, `pnpm verify`
- 手動チェック: アバターの動作スモークテストには、壊れた指がないこと、顔のちらつきがないこと、古いフレームの回帰がないことが含まれます。
- ベンチマークまたはレイテンシーの証拠: `docs/benchmarks/quality-gates.md`を参照してください。

## 検討された代替案

### 生のランドマークを直接マッピングする

- 利点: 実装の手間が最も少ない。
- 欠点: 揺れ、NaN値、左右の入れ替え、アバターリグに対して不可能なポーズを露呈します。

### マッピングの前に安定性レイヤーを要求する

- 利点: 予測可能なアバターの動作とテスト可能な品質ゲート。
- 欠点: わずかなレイテンシーと複雑さのコスト。

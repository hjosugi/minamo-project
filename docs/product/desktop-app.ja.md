# Minamo Studio デスクトップアプリ

Minamo Studio は、既存のトラッカー、ビューワー、リプレイツールのための Tauri デスクトップシェルです。

## コマンド

```sh
pnpm desktop:check
pnpm desktop:dev
pnpm desktop:build
```

`pnpm desktop:dev` はネイティブコントロールサーフェスを開きます。**アバターを開く**を選択し、OSファイルピッカーで Inochi2D の `.inp`/`.inx` または VRM の `.vrm`/`.glb` ファイルを選択すると、Minamo がビューワーを開き、すぐに読み込みます。パッケージ化されたビルドはデフォルトで `src-tauri/target/release/bundle/` に書き込まれます。

## ネイティブアバターの読み込み

デスクトップシェルは選択された標準ファイルパスを Rust 内に保持し、名前、フォーマット、バイト長、およびプロセスローカルのリビジョンのみをウェブビューに送信します。ビューワーは生のバイナリ IPC レスポンスを通じて一致するリビジョンを要求し、ファイルドロップで使用される同じレビュー済みアバターローダーを介してメモリから読み込みます。

- サイズ制限は 256 MiB です。空のファイル、サポートされていないファイル、非正規ファイルは読み込み前に失敗します。
- 選択状態は現在のアプリプロセスのみに持続し、永続化されません。
- ウェブビューは任意のパスを送信できず、広範なファイルシステムやアセットプロトコルのスコープは付与されません。`main` と `viewer` ウィンドウのみがコア IPC 機能を受け取ります。
- ブラウザプレビューは引き続きサポートされます。Tauri ランタイムが接続されていない場合は、ビューワーファイルコントロールを使用するか、ドラッグアンドドロップを行ってください。

## バンドルされたページ

- `desktop/` はコントロールサーフェスを開きます。
- `tracker/` はバンドルされたウェブアセットからウェブカメラトラッキングを実行します。
- `viewer/` はバンドルされたウェブアセットからアバターをレンダリングします。
- `replay/` はバンドルされたウェブアセットからローカル KGM1 JSONL 録画を公開します。

## 仮想カメラバックエンド

デスクトップシェルはプラットフォームバックエンドの状態を報告します。ビューワーフレームのストリーミングは、各 OS が異なる署名されたドライバーまたは拡張パスを必要とするため、意図的にネイティブバックエンドの境界の後ろにあります。

| OS | バックエンドターゲット | 現在の状態 |
|---|---|---|
| Linux | `v4l2loopback` | ドライバーと最初の `/dev/video*` 検出がデスクトップシェルで行われました |
| Windows | Media Foundation ソフトカメラ | バックエンドブリッジがインストールされていません |
| macOS | CoreMediaIO カメラ拡張 | 拡張がインストールされていません |

出力バックエンドの出荷基準：

- フレームを書き込む前に明示的なユーザーのオプトイン
- OS ごとに 1 つの会議アプリでの可視デバイス
- OBS ブラウザソースはドライバーフォールバックとして利用可能
- デスクトップステータスパネルはバックエンド、デバイス、および状態を報告します

Linux、Windows、macOS の 1 つの会議アプリでビューフレームが仮想カメラとして表示されるまで、問題 KGM-050 をオープンに保ってください。

# トラッカーとしての電話モード

ステータス: QR/トークンのランタイムが #226 に実装されました; 機能ベースの WT-to-WSS
交渉が #227 に実装されました; 実際のブラウザ/iPhone のタイミング証拠 (#228)
は保留中です。

## 目標

デスクトップビューアまたは OBS ソースが他の場所で実行されている間、電話ブラウザをトラッカーカメラとして使用します。

## URL 契約

デスクトップアプリまたはランディングページは、次のための QR コードを生成できます:

```text
/tracker/?mode=wt&room=<room>&token=<token>&wtUrl=<wt-relay>&wsUrl=<wss-fallback>&camera=user
```

オプションのパラメータ:

- `wsUrl` はデスクトップペアリングインターフェースによって選択された WebSocket リレーのため
- `wtUrl` と `wtHash` は WebTransport ルームのため
- `resolution=480p|720p|1080p`
- `fps=30|60`
- `mirror=0|1`
- `camera=user|environment` は前面/背面カメラの優先設定のため

トラッカーは、コントロールをレンダリングする前にこれらの値を適用します。トークンを解析後に可視のアドレスバーから削除し、QR トークンをローカルストレージに保持せず、ページ内トークン入力をパスワードフィールドとしてレンダリングします。ビューアは同じ `room`、`token`、および `wsUrl` 契約に従います。

## デスクトップペアリングフロー

1. `relay-node` を起動し、Minamo Studio を開きます。
2. **トラッカーをペアリング** で、電話が到達できるトラッカー URL を設定します。実際の電話には HTTPS LAN ホスト名/アドレスを使用します; `localhost` は電話自体を意味し、デスクトップには到達しません。
3. 一致する WebSocket リレー URL、ルーム、前面/背面カメラ、解像度、フレームレート、およびトークンの有効期限を設定します。HTTPS 電話 URL の場合、リレーは `wss://` でなければなりません。オプションで WebTransport を有効にし、その `https://` URL と証明書ハッシュを提供します。
4. QR を生成してスキャンします。トラッカー URL とビューア URL は独立してコピーすることもできます。可視のフォールバックテキストはトークンが赤actedされており、クリップボードの値にはライブトークンが含まれています。
5. **QR を再生成** を使用して前のトークンを原子的に無効にし、置き換えを発行するか、**今すぐ期限切れ** を使用して今後の参加を即座に拒否します。

デフォルトのトークンの有効期限は5分です（許可される範囲: 30秒から15分）。`relay-node` は発行されたトークンをメモリに保持し、各トークンを1つのルームにバインドし、期限切れまたは取り消されたトークンを WebSocket コード `4401` で拒否し、トークン API レスポンスに `Cache-Control: no-store` を送信します。既存の
`MINAMO_RELAY_TOKEN` デプロイメントは引き続きサポートされます。

## セキュアなトランスポート交渉

- トランスポートの選択は、ユーザーエージェント文字列ではなく、ランタイムの機能を使用します。
- WT が構成されていて `WebTransport` が存在する場合、トラッカー/ビューアは WT を試みます。
- サポートされていない WT、セットアップの失敗、タイムアウト、または接続の失敗は、構成された WSS エンドポイントにフォールバックします。
- HTTPS ページでは、プレーンな `ws://` は実行可能な混合コンテンツエラーで拒否され、ローカルの BroadcastChannel は隠れたリモートフォールバックとして使用されません。
- ステータス診断は選択されたモードを特定し、赤actedされた失敗理由を保持します。ルームトークンは表示されたエラーから削除されます。

これにより、Safari 26.4+ は WT を試みることができ、Safari 26.3 およびそれ以前は WSS を使用します。デバイス/ブラウザのマトリックスと5回のコールドタイミングランは #228 の証拠作業として残ります; ランタイムサポートのみでは実際のデバイスの PASS ではありません。

## 要件

- LAN 上の HTTPS、`docs/DEV_HTTPS.md` のセットアップを使用。
- ビデオのアップロードはありません。電話は KGM モーションフレームのみを公開します。
- ローカルモードの外でルームトークンが必要です。
- ビューア URL は独立してコピーできるため、電話は OBS アクセスを必要としません。
- ペアリング QR スクリーンショットと診断は、問題に添付される前に QR/トークンを赤actedする必要があります。リレーは通常のログにトークン値を書き込みません。

## 実装ノート

既存のトラッカーはブラウザ専用のままです。QR 生成は Vite/Tauri デスクトップバンドル内のピン留めされた `qrcode` パッケージを使用します。リレーはルームの資格情報のみを発行し、KGM モーションフレームを転送します; 生のビデオ URL またはアップロードルートは導入されません。

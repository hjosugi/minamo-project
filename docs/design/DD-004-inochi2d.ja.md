# DD-004: Inochi2D レンダーバックエンド (inox2d WASM)

ステータス: 実装済み (実験的)。バックログ: KGM-038。ランタイムの問題: #229。

## 問題

2D VTuber エコシステムは、プロプライエタリフォーマット (Live2D) に支配されています。
Inochi2D はオープンな代替手段 (パペットフォーマット .inp/.inx) であり、inox2d はその Rust レンダラーです。
ブラウザはまだ快適に動作しない唯一の場所であり、Minamo は一級のオープン 2D パスを望んでいます。

## 目標

- ビューアで Inochi2D パペットを 60 fps で読み込み、レンダリングすること。
- KGM チャンネルからパペットパラメータを VRM と同じマッピングエディタを通じて駆動すること (KGM-044): 1 つのマッピングモデル、2 つのバックエンド。

## 実装されたアプローチ

1. 公式の inox2d WebGL2/glow レンダラーを wasm-bindgen を介して wasm32 にコンパイルします。
   ソース依存関係は、アップストリームコミット `df8413e6b0c525dbb880b4dca2bdf0a5d4b9aaba` に固定されています。
   グルーと 580 KiB の WASM アーティファクトは、オフラインの Viewer/Tauri ビルド用にチェックインされています。
2. 薄い JS ラッパーが `src/adapters/inochi2d_mapper.ts` の `Inochi2DRuntimeAdapter` に一致します: `load(bytes)`, `setParam(name, v)`, `update(dt)`, `render(target)`, `listParams()`, `dispose()`。
3. パラメータ発見は、`.inp` と `.inx` で共有される長さプレフィックス付きの JSON ヘッダーのみを解析します; テクスチャバイトは JSON としてデコードされません。
   名前付きパラメータ、ベクトル形状、範囲、およびデフォルトが既存のエクスプレッションマッピングエディタに供給されます。
4. Inox2D は、アルファとステンシルを持つ隠れた WebGL2 キャンバスにレンダリングします。
   Three.js は、そのキャンバスを 1 つの `CanvasTexture` プレーンを通じてアップロードし、既存の Viewer レンダラーを唯一の可視スワップチェーンとして残します。
   ソーステクスチャは、各フレームの最終シーンレンダリングの前にリフレッシュされます。
5. ファイル入力、ドラッグ＆ドロップ、および `?inochi=<cors-url>` は、`.inp`/`.inx` 拡張子によってこのバックエンドを選択します。
   アバターを置き換えたり、ページをアンロードしたりすると、WASM オブジェクトが解放され、Three リソースが破棄され、隠れた GL コンテキストが失われ、隠れたキャンバスが削除されます。

## デフォルトマッピングヒューリスティックス

| KGM ソース | Inochi2D パラメータ (名前一致、ファジー) |
|---|---|
| ヘッド ヨー / ピッチ / ロール (クォータニオンから) | ヘッド ヨー / ピッチ / ロール |
| アイ ブリンク 左/右 | アイ L/R ブリンク |
| ジョー オープン | マウス オープン |
| マウス スマイル 平均 | マウス スマイル |
| ポーズ ショルダー ロール | ボディ ロール |

正規化されたパラメータ名によるファジーマッチング; 一致しないものは、推測するのではなく、エディタで未マッピングとして表示されます。

ランタイムは、発見されたパラメータ名が既知のエイリアスと一致する場合にのみファジーなデフォルトを適用します。
一致しないパラメータは、エディタで可視かつ未マッピングのまま残ります。エクスポートは、VRM と同じ `minamo.expression-map.v1` フォーマットを使用します。

## 診断とサポートされていない機能

- 無効なマジック、切り捨てられた JSON、形式が不正なパペットデータ、欠落した WebGL2/ステンシル、およびサポートされていないレンダラー機能には、別々のユーザー向け診断があります。
- 固定されたアップストリームパーサーは、`.inp` と `.inx` の両方をサポートしています。
  BC7 パペットテクスチャはサポートされていないため、ユーザーはそれらのテクスチャを PNG または TGA として再エクスポートする必要があります。
- Inox2D は、その Rust レンダラーを実験的と説明しています。
  アップストリームの機能ギャップは、サイレントボット/VRM フォールバックを引き起こしません。

## リスク

- Inox2D WASM の成熟度: 固定されたソースと生成されたアーティファクトは、アップストリームを更新する際に一緒に再レビューされる必要があります。
- 2 つの GL コンテキストが必要ですが、Three.js コンテキストのみが画面に表示されます。
  Inox2D コンテキストは内部テクスチャプロデューサーです。
- 再配布可能なリアルパペットのビジュアル/レイテンシマトリックスは #230 によって追跡され続けます; #229 はランタイムとブラウザライフサイクルをカバーし、その別の証拠は含まれません。

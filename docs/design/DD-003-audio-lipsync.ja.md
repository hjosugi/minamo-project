# DD-003: オーディオリップシンクフュージョン

ステータス: デザイン。バックログ: KGM-045, KGM-046。

## 問題

視覚的な口の追跡は、ストリーマーが必要とする正確なタイミングで失敗します: 低照度、顔が部分的にフレーム外、カメラが30 fps未満。オーディオは常にあなたが話しているときに知っています。VSeeFaceクラスのツールはオーディオリップシンクをフォールバックスイッチとして扱いますが、Minamoは両方を継続的に融合させるべきです。

## 目標

- マイク -> ビジュアルモーフィームの重みを完全にデバイス上で処理し、< 80 msのオーディオからアバターへの遅延。
- フュージョン、切り替えではなく: オーディオがオープンネスのタイミングを駆動し、ビジョンが形状を駆動します。
- ゼロ設定のデフォルト; ゲインと感度のための高度なパネル。

## ビジュアルモーフィーム推定オプション

1. フォルマントヒューリスティック (v1): AudioWorkletがRMSエネルギー + 最初の2つのフォルマント (LPCまたはバンドエネルギー) を計算します。{aa, ih, ou, ee, oh} の重みにマッピングし、小さな決定面を持ちます。安価で堅牢、母音に対して言語に依存しない; 子音はエネルギーの低下を通じて閉鎖として読み取られます。
2. 小型ML (v2): ログメルフレーム上の小さなCNN/GRU (オープンライセンスのビジュアルモーフィームモデル、またはそれを蒸留する) をONNX Runtime Web (wasm EP; モデルは小さい) を介して使用します。より良い子音、約2 ms/frameの予算。

(1)を最初に出荷; (2)は同じインターフェースの背後に。

## フュージョンルール

a = オーディオオープンネス [0,1]、v = ビジュアル顎の開き、c_v = ビジュアル信頼度 (KGM-015からの追跡品質)、VAD = スピーチ確率。

- オープンネス = VAD * max(a, v * c_v) + (1 - VAD) * v * c_v
- 形状 (aa/ih/ou/ee/oh比): c_vが高いときは視覚的、低いときはオーディオ比、c_vに基づく線形ブレンド。
- アタックは速く、リリースは遅く (アタック30 ms、リリース120 ms) して、破裂音が登録され、ノイズで口がパタパタしないようにします。

ビューワー側またはトラッカー側で実行されます; デフォルトはトラッカー側で、融合された結果がワイヤに送信されるものです (KGMはレンダラーに依存しません)。5つのビジュアルモーフィームチャンネルを追加するか、既存の口のチャンネルを再利用する必要があります; 決定: 既存のARKitチャンネル (jawOpen, mouthFunnel, mouthPucker, mouthStretch*) を再利用して、プロトコルが変更されないようにします。

## プライバシー

オーディオはデバイスから出ることはありません; 派生された口のチャンネルのみが出ます。マイクが初めて有効になったときに、これをUIで明示的に示します。

## 声のアクセント

声のアクセントはデフォルトでオフで、同じローカル専用のマイクパスを共有します。有効にすると、VADエネルギーは限られた眉の微小な上昇と、KGMエンコーディングの前にサブディグリーの頭のピッチアクセントを追加します。サイレントなVADレベルはアイデンティティの重みとゼロの頭の振り幅を返すため、静寂は余分な動きを生み出しません。

## オーディオリップシンク実装

オーディオリップシンクは、ローカルマイクパスを再利用し、クラウドASRを使用しないオプトインのトラッカーフィーチャーです。AudioWorkletはRMSと低/中/高エネルギーバンドから毎20 msごとにビジュアルモーフィームフレームを投稿します。トラッカーはそれらのフレームを既存のARKit口のチャンネルに融合させ、オーディオがjawOpenのタイミングを駆動し、ビジュアル信頼度がどれだけオーディオの形状の詳細がmouthFunnel、mouthPucker、mouthStretch*を満たすかを決定します。80 msのターゲットより古いフレームは遅れて再生されるのではなく無視されます。

## リスク

- AudioWorklet + getUserMediaの許可フローはUXの摩擦を追加します: マイクは厳密にオプトインであり、トラッカーはそれなしで機能します。
- キーボード/デスクのノイズがVADをトリガーする: エネルギーゲート + スペクトルフラットネスチェック。

# DD-007: 録音と再生 (.kgmファイル)

ステータス: 設計中。バックログ: KGM-047, KGM-048。

実装ノート: ブラウザは、コンパクトな `.kgm` コンテナと
改行区切りの `minamo.kgm1.motion-jsonl.v1` デバッグファイルの両方を録音します。ビューワーと再生ページは、ドロップされた `.kgm`、`.jsonl`、および `.ndjson` ファイルを読み込むことができます。

## 動機

同じ機能の3つの利用者:
1. ユーザー: テイクを録音し、それを再生し、モーションをエクスポートする (VRMA)。
2. 開発: 録音されたセッションは、ソルバー/コーデック/レンダリングの変更に対する決定論的回帰フィクスです。
3. KGM2 (DD-006): 圧縮の主張を検証するためのコーパスが必要です。

## ファイル形式

コンテナ、リトルエンディアン:

```
magic "KGMR" u32 | version u8 | flags u8 | reserved u16
meta_len u32 | meta JSON (utf-8): { created, tracker: {backend, versions},
                                    camera: {w, h, fps}, notes }
その後、繰り返されるレコード:
  dt_ms u16 (前のレコードからのデルタ) | len u16 | フレームバイト (KGM1/2)
```

フレームは、送信された通りにそのまま保存されるため、再生は
正確なデコードパスを実行します。60 fps の顔のみ: 80 バイト/レコード -> 圧縮されていない場合、10 分あたり約 4.6 MB; gzip (ダウンロード時の CompressionStream 経由) でおおよそ半分になります。

## トラッカー UI

録音 / 停止; ライブサイズカウンター; Blob 経由でダウンロード。録音は
transport.send の直前にエンコードされたバッファをタップするため、トランスポートの状態は
無関係です。

## ビューワー再生

.kgm ファイルをドロップ: トランスポートのようなソースが dt
スケジュールに従ってレコードを再生し (速度制御付き)、同じフレームアプリケーションパスに供給します。
JSONL MVP は、録音された顔、ポーズ、および手のフィールドを解析した後、同じパスを使用します。ループトグル。スクラブバーは v2 (キーフレームインデックスが必要; KGM2 キーフレームが存在すれば簡単です)。

## VRMA エクスポート (KGM-048)

VRM アプリケーションレイヤーを通じて再生し、固定 dt で正規化されたボーン
回転と表情値をサンプリングし、次に VRM アニメーション glTF にシリアライズします。エクスポートはビューワー内で実行されるため、エクスポートはアクティブな
マッピングを反映します (KGM-044)。エクスポート前にトリムイン/アウトポイントを設定します。

実装されたエクスポータは、`VRMC_vrm_animation` 拡張を持つバイナリ `.vrma` GLB を出力します。再生ページはトリム開始/終了フィールドとループマーカーを公開します。エクスポートされたクリップには、必要なヒューマノイドボーンマップ、頭部ヒューマノイドボーン回転キーフレーム、およびスカラーウェイトを格納する翻訳 X を持つプリセット表情ノードが含まれています。表情ウェイトは、aa/ih/ou/ee/oh、まばたき、笑顔/幸せ、怒り、驚き、そして中立のプリセットに対してエクスポートされます。

## テスト使用

`fixtures/` には、1つのコミットされた30秒のセッションがあります。CI はそれをデコード -> ソルブ -> マッピングを通じて再生し、要約統計 (チャネルごとの平均/分散、最終ポーズ) が許容範囲内であることを確認し、カメラなしで回帰をキャッチします。

# DD-006: KGM2 プロトコル

ステータス: 参照実装。バックログ: KGM-027, KGM-028, KGM-029。
何も置き換えず、KGM1は有効のままで、リレーはどちらも理解する必要はありません。

## 動機

KGM1は意図的に単純です: 固定レイアウト、各フレームの絶対値。手（DD-001）やより豊富なボディデータにより、フレームごとのクォータニオン数が増加し、長いアイドル時間がバイトを無駄にします。KGM2は、コアプロパティを維持しながら、約2倍の削減を目指します: 損失は安価です。

## 変更点

### 1. 最小の3つのクォータニオン (KGM-027)

単位クォータニオンの最大成分は、他の3つから復元可能です。エンコード: 最大成分インデックスのための2ビット + 残りの成分のための3 x 10ビット符号付き、1/sqrt(2)でスケーリング。合計32ビット、KGM1の64ビットに対して。最大角誤差は約0.3度で、ウェブカメラのトラッキングノイズを下回ります。

### 2. キーフレームを持つデルタフレーム (KGM-028)

- Kフレームごとにキーフレーム（デフォルトK=30、つまり60 fpsで1秒間に2回）:
  絶対KGM2フレームと同一です。
- デルタフレーム: 最後のキーフレーム（前のフレームではなく）に対する各チャネルの符号付きデルタをi8で、2倍の範囲スケールで。i8範囲を超えるチャネルは、そのチャネルを絶対値のエスケープリストに強制します。
- デルタを最後のキーフレーム（前のフレームではなく）に基づけることで、間隔内の損失は何も劣化させません: 各デルタは、そのキーフレームが到着した後に独立してデコード可能です。キーフレームの損失は次のものまで劣化します（<= 500 ms）、輸送の信頼性クラスに一致します。
- ヘッダーの増加: u16キーフレームID; デルタフレームは明示的にそれを参照するため、デコーダは間違ったベースにデルタを適用することはありません。

### 3. スパースチャネルマスク (KGM-029)

52のフェイシャルチャネルに対する7バイト（56ビット）の存在マスク; リストにないチャネルはそのキーフレーム値を保持します。デルタと組み合わせることで、アイドルフェイスフレームはヘッダー + マスク + 数バイトになります。

### 4. 予約ブロック

ブロックビット2-5は予約されています: HAND (DD-001)、GAZE (KGM-016 真の視線ベクトル)、SOURCE (コラボルームのu16ソースID、KGM-043)、AUX。

## フォーマットスケッチ

```
header:  magic u16 | ver=2 u8 | blocks u8 | t u32 | seq u16 | kf_id u16
FACE-K:  quat s3 u32 | pos i16x3 | weights u8x52
FACE-D:  quat s3 u32 | pos delta i8x3 | mask 7B | deltas i8xN | esc u8 + pairs
```

現在のJS参照は`blocks`を`frame_type`バイトに絞ります: `1 = キーフレーム`、`2 = デルタ`。デルタフレームは固定の7バイトチャネルマスクを使用し、エスケープリストを省略します; i8範囲外のチャネルデルタはキーフレームを強制します。これにより、最初に出荷されるプロファイルはシンプルで決定論的でありながら、同じワイヤーレベルの回復ルールを保持します。

## 互換性

バージョンバイトがすべてを制御します。エンコーダはセッションごとにKGM1またはKGM2を出力できます; ビューアは両方をデコードします。JS参照からのゴールデンベクトル（KGM-031）は、バージョンが出荷される前にフォーマットを固定します。

## 検証ターゲット

- 最大最小3つのクォータニオン角誤差 < 0.5度、1Mのランダム回転に対して。
- JS最小3つのエンコード+デコード < 1μsごとのクォータニオン。
- 記録されたコーパスに対して平均35%以上のサイズ削減 (KGM-047フィクスチャ)。
- 10%の均一な損失は1つのキーフレーム間隔内で回復します。

## 実装証拠

- コーデック: `shared/kgm2.js`
- テスト: `tests/run-tests.mjs`
- 参照コーデックノート: `docs/transport/kgm2-reference-codecs.md`

回帰スイートは、1Mクォータニオン精度ゲート、タイミングされたJSエンコード/デコードループ、スパースチャネル保持セマンティクス、30バイト未満のアイドルデルタ、ベースキーフレームなしのデルタの拒否、次のキーフレーム間隔でのドロップされたキーフレーム後の回復を検証します。

# キャリブレーションデザイン

## 1. 目標

キャリブレーションは、安価なウェブカメラをより良く感じさせます。

## 2. キャリブレーションモジュール

- カメラのミラーと利き手
- 顔のニュートラル
- 目の範囲
- 口の範囲
- 手のスケール
- 指のカール範囲
- アバターリグの制限
- ドラムキットのゾーン
- 音声/映像の同期オフセット

## 3. 実装された顔のフロー

トラッカーには30秒のガイド付き顔キャリブレーションがあります：

1. ニュートラルホールドを3秒間。
2. 6つの表情範囲をそれぞれ4.5秒で通過。
3. チャンネルごとのオフセットはニュートラルサンプルから推定されます。
4. チャンネルごとのゲインは表情のピークから推定されます。
5. 生成されたプロファイルはローカルに保存され、JSONとしてエクスポート/インポートできます。

プロファイルは生信号のサニタイズ後、One Euroフィルタリングの前に適用されるため、ライブメーターと出力されるKGM1フレームの両方がキャリブレーションされた信号を使用します。

手動調整は52チャンネルメーターで利用可能です：チャンネル行をドラッグしてゲインを0-2倍に設定し、選択したチャンネルのデッドゾーンを0-0.2に調整し、右クリックまたはタッチの長押しでミュートします。これらの値は同じローカルJSONプロファイルに保存され、エクスポート/インポートを生き残ります。

視線キャリブレーションは、10秒の5ポイント虹彩パスを追加します：中央、左、右、上、下。MediaPipeの虹彩ランドマークが利用可能な場合、Minamoは虹彩の中心と目の輪郭を比較して視線を推定し、保存された中心/スケール補正を適用し、視聴者の視線パスで消費される既存の目の視線チャンネルに結果を投影します。虹彩ランドマークが欠けている場合、元のブレンドシェイプの視線をそのままにします。

## 4. UX

UIは、進捗バーとライブ品質警告を使ってユーザーを短いアクションに導くべきです。

例：

- 「手を開いてください」
- 「拳を作ってください」
- 「2回まばたきしてください」
- 「あいうえおと言ってください」
- 「スネアを4回叩いてください」
- 「ハイハットを4回叩いてください」

## 5. 保存されたプロファイル

キャリブレーションプロファイルはローカルファーストのJSONであるべきです。

```json
{
  "schema": "minamo.calibration.v1",
  "name": "guided-2026-07-06T00:00:00.000Z",
  "createdAt": "2026-07-06T00:00:30.000Z",
  "offsets": [0.0],
  "gains": [1.0],
  "deadzones": [0.0],
  "muted": [false]
}
```

# Minamoアーキテクチャ

## 目標

通常のウェブカメラとブラウザを持っている誰もが、低遅延かつほぼゼロの帯域幅で2Dまたは3Dアバターを操作できるようにします。iPhoneは不要、深度センサーも不要、インストールも不要です。

## 原則

1. **エッジでの推論。** カメラの画像はユーザーのマシンから出ることはありません。動きのパラメータだけが送信されます。これはプライバシーの特性であり、同時に帯域幅の特性でもあります。
2. **動画ではなくパラメトリックモーション。** トラッキングされた顔はフレームあたり約76バイトです（docs/PROTOCOL.mdを参照）。パラメータを送信し、どこでもレンダリングします。
3. **リアルタイム状態のための信頼性のない輸送。** 遅れたトラッキングフレームは無駄なトラッキングフレームです。WebTransportのデータグラムは再送信するのではなくドロップされるため、最悪の遅延が制限されます。
4. **モデルに依存しないプロトコル。** KGM1は標準的なARKit-52チャネルセットを運びます。トラッカーのバックエンド（現在はMediaPipe、明日はYOLO-poseまたはRTMPose）は、レンダラーや中継に触れることなく変更できます。

## レイテンシ予算（ターゲット、ローカルネットワーク）

| ステージ | 予算 |
|---|---|
| カメラキャプチャ + キュー | 8-16 ms |
| Face Landmarker推論（GPU） | 6-12 ms |
| ソルブ + フィルター + エンコード | < 1 ms |
| WebTransportデータグラム、LAN | < 2 ms |
| ビューアの緩和 + レンダリング | 8-16 ms |
| **ガラスからガラスまで** | **約25-45 ms** |

One Euroフィルタリングは、スローモーション中のみ効果的な遅延を追加しますが、まさにその時に遅延は目に見えません。これが標準的な選択肢である理由です。

## なぜこれらのコンポーネントなのか

- **MediaPipe Face Landmarker**は、現在利用可能な最も強力なオープンなウェブカメラ顔モデルです：52のARKit互換のブレンドシェイプとメトリックヘッド変換を持ち、WASMを介して統合GPUで60 fpsで動作します。これは、Kalidokitが手動で行わなければならなかった古いランドマークからブレンドシェイプへのヒューリスティックを置き換えます。
- **WebTransport (QUIC)**は、ブラウザにデータグラムを提供します。WebSocketはフレームをドロップできません。TCPの再送信は、失われたパケットを遅延スパイクに変えてしまいます。Rustの中継（`wtransport`）は、ファンアウトパスの割り当てを軽量かつ予測可能に保ちます。
- **中継にはRustを使用**しています。中継は純粋なI/Oファンアウトであり、予測可能な尾遅延が開発者の速度よりも重要です。
- **スケールアウトにはErlang/Elixirを使用**しています（設計文書DD-005）：1つのRust中継は1人のストリーマーには十分です。地域を超えて数千の視聴者にファンアウトすることは分散問題であり、接続ごとのBEAMプロセスと分散pub/sub（Phoenix.PubSub）がそれに対する実証済みの形です。KGM1ペイロードは不透明なバイトのままであり、BEAMはルーティングのみを行います。
- **three-vrm**は、スプリングボーンとVRM表現システムを使用してVRM 0.x/1.0をレンダリングします。2Dパス（Inochi2Dはinox2d WASMを介して）は設計文書DD-004です。

## 圧縮戦略

2つの異なる問題：

1. **モーション（連続、ホットパス）。** 量子化のみ：i16クォータニオン、u8重み、mm位置。76バイトではエントロピーコーディングはCPUの価値がありません。KGM2は最小の3つのクォータニオンパッキングとデルタフレームを追加します（DD-006）。
2. **アバターアセット（一度きり、コールドパス）。** VRM/glTFメッシュはmeshoptimizer（EXT_meshopt_compression）またはDracoで圧縮され、テクスチャはKTX2 / Basis Universalで圧縮されたままGPUに保持されます。これがアセット配信パイプラインの計画です（バックログKGM-041）。

## 以前のアートとの関係

| プロジェクト | 何をするか | Minamoが取る/変更すること |
|---|---|---|
| Kalidokit | MediaPipeランドマークからリグパラメータへのJSソルバー | ソルバー層として同じ役割；ヒューリスティックソルビングの代わりに新しいFace Landmarkerのブレンドシェイプ出力を直接使用 |
| OpenSeeFace / VSeeFace | Python UDPトラッカー + Unityレンダラー | 同じ「ワイヤ上のパラメータ」アイデア；MinamoはUDPをWebTransportに置き換え、ブラウザから動作するようにします |
| VTube Studio | 商業用2Dトラッキングアプリ | キャリブレーションUXと品質基準の参考 |
| Inochi2D / inox2d | 2Dパペットフォーマット + Rustレンダラー | 計画中の2Dレンダーバックエンド（DD-004） |
| moeru-ai/airi | AI VTuberコンパニオンスタック | アバターを操作するためにKGM1フレームの潜在的な消費者 |
| handcrafted-persona-engine | ペルソナ/アバターエンジン | 同じ；KGM1はこのようなエンジンによって簡単に消費できるように設計されています |

# 品質ゲート

## P0ゲート

- NaN/Infinityがレンダラーに到達しない
- 不可能な指のポーズがクランプ警告なしにレンダラーに到達しない
- 閉じた口のニュートラルテスト中にしきい値を超える口のちらつきがない
- 開いた目のテスト中にしきい値を超えるまばたきのちらつきがない
- 一つの視覚フレームからのみドラムヒットが発生しない
- デフォルトモードでの生のウェブカメラアップロードがない

## P1ゲート

- 顔と手のための現代のノートパソコンでの60fpsターゲット
- 低スペックのノートパソコンでの30fpsフォールバック
- 500msの遮蔽後の安定した手の再取得
- シンプルなスネアヒットのためのターゲットしきい値以下のヒットタイミングエラー
- VRMおよび1つの2Dフォーマットのためのアバターマッピングテスト

## P2ゲート

- ターゲットレイテンシー以下のWebTransportモーションストリーミング
- モバイルブラウザのスモークテスト
- 低照度品質警告
- カスタムスティック検出器ベンチマーク

## 実行時品質スコア

トラッカーは正規化されたサブスコアからフレームごとのスコアを計算します：

| 入力 | 重み | 良好なフィクスチャの期待値 |
| --- | ---: | --- |
| 平均ルマ | 0.22 | 通常の屋内照明は決して `poor` ではない |
| ランドマーク信頼度 | 0.28 | フルフェイスが見える状態は `good` のまま |
| 観測されたfps | 0.18 | 30-60 fpsは使用可能のまま |
| 推論時間 | 0.14 | 短いスパイクは劣化するが回復する |
| ローリングドロップフレーム | 0.10 | スタートアップのスタールは後のフレームを悪化させない |
| モーションブラー推定 | 0.08 | 高速な動きは独立して警告できる |

フィクスチャは良好な屋内照明、低照度、ドロップfps、高い推論時間、遮蔽、およびモーションブラーをカバーする必要があります。品質の退行は手動カメラテストに到達する前にユニットテストで失敗するべきです。

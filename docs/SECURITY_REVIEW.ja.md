# セキュリティレビューチェックリスト

カメラインプット、ローカル録音、リレー、モデルアセット、WebTransport/WebSocket、トークン、またはリリースアーティファクトに関わる機能をマージする前に、このチェックリストを使用してください。

## データ境界

- カメラフレームはデフォルトでトラッカーページから出ません。
- 生の音声はデフォルトでローカルデバイスから出ません。
- リレーペイロードにはKGMモーションデータのみが含まれ、ビデオや音声は含まれません。
- メディアを送信する機能は、明示的なユーザーアクションと目に見えるUIコピーを必要とします。
- JSONL録音は明示的にローカルであり、ユーザーが開始します。

## リレーとトランスポート

- オプションのルームトークンは、共有または公開リレー用に有効化されています。
- `MINAMO_ALLOWED_ORIGINS`は公開WebSocketデプロイメント用に設定されています。
- トークンチェックは定数時間比較を使用します。
- 新しいネットワーク機能はダウングレード、リプレイ、およびオリジンの動作を文書化します。
- 古いまたは順序が入れ替わったフレームは、シーケンスを認識する受信者ロジックによって拒否されます。
- WebTransport証明書フィンガープリントは、ローカル開発中に明確に表示されます。

## クライアントストレージ

- キャリブレーションプロファイル、トラッカー設定、および録音は、ユーザーがエクスポートしない限りローカルに留まります。
- ローカルストレージキーには秘密が含まれません。
- インポートパスは、値を適用する前に形状と範囲を検証します。
- エクスポートされたファイルは、機能が明示的にそれらを録音しない限り、生のカメラフレームを避けます。

## 電話ペアリングトークン

- `relay-node`は生成されたペアリングトークンを1つのルームにバインドし、そのTTLを30〜900秒に制限し、期限切れ、取り消し、未知、または間違ったルームのトークンを拒否します。
- 再生成は以前のトークンを取り消します。発行およびQRフォールバックエンドポイントは`Cache-Control: no-store`を使用し、`MINAMO_ALLOWED_ORIGINS`を尊重します。
- トラッカー/ビューページは、解析後にアドレスバーからQRトークンを削除し、それらの一時トークンをローカルストレージに保持しません。リレーログにはトークンやQRペイロードの値は含まれません。
- QR画像とコピーされたURLは資格情報です。証拠のスクリーンショットや診断は、公開前にそれらを削除しなければなりません。
- 公開電話ペアリングは、HTTPS/WSSおよび構成されたオリジン許可リストを必要とします。#227はHTTPSからのプレーンWSを拒否し、ランタイムがWebTransportを公開する場合にのみ構成されたWTエンドポイントを優先し、WSSにのみフォールバックします。ローカルモードはリモート電話ペアリングのためのサイレントフォールバックには決してなりません。トークンUIはTLSの代替にはなりません。

## サプライチェーン

- サードパーティのモデルおよびWASMアセットには、実用的な場合にピン留めされたバージョンとSHA/SRIメタデータがあります。
- 新しい依存関係は`DEPENDENCY_POLICY.md`に従います。
- CDNフォールバックは文書化されており、ローカルベンダリングパスが優先されます。
- ドキュメント、ソース、イシュー本文、ログ、または録音に秘密はコミットされません。

## リリースレビュー

- 完全な`RELEASE_CHECKLIST.md`を実行します。
- 公開前に生成されたアーティファクトをレビューします。
- 公開デプロイメントドキュメントには、トークンおよびオリジンの構成が含まれます。
- セキュリティに関連する変更は、イシューのクローズノートに残存リスクを言及します。

# KTX2 テクスチャ圧縮ガイド

ステータス: issue #158 のデザインが実装されました。関連: #41。

KTX2/BasisU はテクスチャを GPU 上で圧縮したまま保持し、ダウンロードサイズと VRAM を削減します。これは、[avatar-compression.md](avatar-compression.md) におけるパイプラインのテクスチャステージです。

## 手順

1. まずテクスチャインベントリを確認します (`pnpm inspect:glb -- avatar.glb`) し、テクスチャの数、寸法、マテリアル参照をメモします。
2. 大きなアルベドおよびノーマルマップを KTX2 に圧縮します（ノーマル/ディテールには UASTC、品質が許す場合は大きなアルベドには ETC1S を使用）。
3. 小さな UI テクスチャ、シャープなアルファマスクテクスチャ、および著作ツールがマテリアル参照を保持できないリグのために、非圧縮の PNG/WebP フォールバックを保持します。
4. アセットの横に GPU メモリとファーストフレームのロード時間を記録します。
5. 出荷前に、three.js の `KTX2Loader` トランスコードターゲットがモバイルブラウザ層でロードされることを確認します。

## リグ破損のリスク

- ノーマルマップに対する攻撃的な ETC1S は、顔や衣服のシマーを引き起こします。ノーマルマップには UASTC を使用してください。
- アルファカットアウトテクスチャのブロック圧縮はエッジをほつれさせ、OBS の透明モードで可視化されます。
- エンコーダが画像スロットを書き換えると、マテリアル参照が間違ったテクスチャを指す可能性があります。ベースラインインスペクションに対してマテリアル数とスロットマッピングを確認してください。

## テスト方法

- テクスチャ受け入れチェックリスト（`evaluateAssetChecklist` に反映）: アルベドグラデーションが保持され、ノーマルマップにシマーがなく、透明モードでアルファエッジがクリーンで、マテリアル参照が intact であり、モバイルトランスコードターゲットがフォールバックのスタールなしでロードされること。
- `pnpm test` はテクスチャの回帰に対するチェックリスト評価者ゲートをカバーします。
- 手動: ニュートラル、まばたき、口を開けたポーズでのサイドバイサイドビューア比較 ([visual-regression-checklist.md](visual-regression-checklist.md))。

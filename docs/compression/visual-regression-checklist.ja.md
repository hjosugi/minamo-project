# アバター視覚回帰チェックリスト

ステータス: イシュー #162 のためにチェックリストを実装しました。関連: #41。

すべての最適化されたアバターは、出荷前に元のアバターと比較する必要があります。
これは、[avatar-compression.md](avatar-compression.md) のテクスチャおよびジオメトリステージで参照される受け入れゲートです。

## 手順

1. 同じビューワーURLから元のアバターと最適化されたアバターをキャプチャします。
2. 以下の各ポーズについて、フルフレーム画像と切り抜いた顔画像を保存します：
   - ニュートラルポーズ
   - 左目を閉じる / 右目を閉じる
   - 口を開ける、笑顔、口を尖らせる
   - 左 / 右 / 上 / 下を見る
   - 頭を回した後のスプリングボーンの動き
   - 透明なOBS背景
   - 低スペックデバイスの負荷テスト
3. ピクセル差分ツールを使用する場合は、背景と髪の先をマスクして、スプリングモーションが顔やマテリアルの回帰を隠さないようにします。
4. アセットの横に各ポーズの合格/不合格を記録します。

## リグ破損のリスク

- 特定の表情（例：口を尖らせる）でのみ現れる回帰は、ニュートラルポーズだけがチェックされると見逃されます。フルポーズリストは必須です。
- スプリングボーンの違いは、フルフレームの差分で顔の回帰を隠す可能性があります。切り抜いた顔の領域も常に差分を取ります。
- 透明モードのエッジアーティファクトは、不透明な背景では見えません。OBSの透明キャプチャが必要です。

## テスト方法

- ポーズリストはサンプルアセットチェックリスト（`shared/compression-checklist.js`）にエンコードされており、`evaluateAssetChecklist`はアセットの回帰記録から必要なポーズが欠けている場合に失敗します。
- `pnpm test`はチェックリスト評価者をカバーします。
- 手動: ポーズグリッドを完成させ、バンドルされたサンプルアセットのリリースノートに合格/不合格の記録を添付します。

# 2D テクスチャアトラス圧縮ガイド

ステータス: イシュー #160 のデザインを実装しました。関連: #41, #38。

2D アバター（レイヤー付き PNG/PSD、Inochi2D）は、レイヤーをテクスチャアトラスにパックします。このガイドは、アトラスを決定論的かつデバッグ可能に保ち、[avatar-compression.md](avatar-compression.md) の 2D セクションを補完します。さらに、[../product/layered-avatar.md](../product/layered-avatar.md) も参照してください。

## 手順

1. 決定論的なアトラスマニフェストを出力します: 安定したレイヤー名、スロット、深さ、元のサイズ、パックされた矩形、および変換原点。
2. 同じ変換原点を共有する静的レイヤーのみをアトラス化します。デバッグを簡単に保つために、口/目のスワップレイヤーは別々に保ちます。
3. 古い GPU 用に 2 の累乗のアトラスオプションを提供しますが、ソース PNG/PSD レイヤーはエクスポート可能に保ちます。
4. 現在のプリセットでアクティブでない表情パックをレイジーロードします。
5. アルファエッジを確認した後にのみ、圧縮テクスチャバリアントを生成します。
6. アトラスの再構築を通じてリグパラメータ名を安定させ、マッピングエディタとプリセットが引き続き機能するようにします。

## リグ破損のリスク

- レイヤーの変換原点を変更する再パッキングは、変形ピボットをシフトさせ、物理/メッシュ変形を壊します。
- スワップレイヤー（口/目）を一緒にアトラス化すると、状態の切り替えがフレーム間で漏れます。
- 再構築中にリグパラメータの名前を変更すると、保存されたマッピングが無効になります
  ([../product/expression-mapping.schema.json](../product/expression-mapping.schema.json))。

## テスト方法

- レイヤー付きアバターマニフェストはスキーマ検証され
  ([../product/layered-avatar.schema.json](../product/layered-avatar.schema.json))
  `pnpm test` でカバーされています。
- サンプルアセットチェックリスト（`evaluateAssetChecklist`）は、2D エリアのパラメータ名の安定性とアルファエッジの受け入れを検証します。
- 手動: アトラスを再構築し、すべてのプリセットが同じパラメータ名にマッピングされていることを確認し、OBS の透明モードでアルファエッジがクリーンであることを確認します。

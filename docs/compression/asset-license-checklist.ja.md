# アセットライセンスチェックリスト

ステータス: イシュー #163 のためのチェックリストを実装しました。関連: #41。

圧縮、リターゲティング、およびスクリーンショットは、多くのクリエイター・マーケットプレイスライセンスの下で修正または派生使用と見なされます。サンプルアセットは、このチェックリストのライセンスがクリアされるまで最適化または公開されません。詳細は [avatar-compression.md](avatar-compression.md) のライセンスセクションを参照してください。

## ステップ

1. バンドルされたアセットまたはサンプルアセットごとに、以下を記録します：
   - ソースURLまたは著者提供のパッケージ
   - ライセンス名とバージョン
   - 再配布が許可されているか
   - 修正/圧縮が許可されているか
   - クレジットが必要か
   - モデル出力またはスクリーンショットをドキュメントで使用できるか
2. ライセンス記録をアセットの横に保存するか、外部サンプルファイルのリリースノートに保存します。
3. 再配布 **および** 修正の両方が許可されるまで、最適化されたサンプルを公開しないでください。
4. 必要な場合は、アセットと一緒にクレジットテキストを保持してください。

## リグ破損リスク

- ライセンス条件はリグリスクではありませんが、再配布不可のアセットを出荷することはリリースのブロッカーです。ライセンスが欠落しているか曖昧な場合は、失敗した検査と同様にハードストップとして扱ってください。
- ドキュメント内のスクリーンショットは、モデルファイルが再配布されない場合でも出力条項に違反する可能性があります。出力/スクリーンショットの条件を別途確認してください。

## テスト方法

- ライセンスフィールドはサンプルアセットチェックリストの一部です
  (`shared/compression-checklist.js`); `evaluateAssetChecklist` は再配布または修正の許可が欠落しているアセットを失敗させます。
- `pnpm test` はチェックリスト評価者のライセンスゲートをカバーします。
- 手動: いかなるバンドルアセットを公開する前に、ライセンス記録が存在し、リリースに添付されていることを確認してください。

# MoQ 配信評価

ステータス: 評価済み。推奨: MoQTの相互運用性とブラウザ向けリレーの展開がCIでテストするのに十分安定するまで、Minamoの製品配信にはノーゴー。

レビューしたソース:

- IETF `draft-ietf-moq-transport-18`, 2026年5月:
  https://datatracker.ietf.org/doc/draft-ietf-moq-transport/
- IETF Media over QUIC ワーキンググループのチャーター:
  https://datatracker.ietf.org/group/moq/about/

## マッピング設計

MoQTはQUICまたはWebTransport上のパブリッシュ/サブスクライブプロトコルです。メディアに依存せず、タイミングメタデータを運ぶことができるため、KGMフレームはオブジェクトモデルに適合します。

提案された名前空間:

```text
minamo/<room>/<source-id>
```

トラック:

| トラック | MoQ 配信 | Minamo ペイロード | ノート |
|---|---|---|---|
| `motion.delta` | オブジェクトデータグラム | KGM2 デルタフレーム | 最新のみ、すぐに期限切れ |
| `motion.keyframe` | サブグループ/ストリーム | KGM2 キーフレーム | 信頼性のある回復ポイント |
| `control` | ストリーム | クロックプローブ、プロファイル変更 | 信頼性があり、順序付き |
| `events` | ストリームまたはデータグラム | ドラムヒット、シーンイベント | イベント損失が許容できない場合のみ信頼性あり |

グループはKGMキーフレーム間隔にマッピングされます。デルタオブジェクトは最新のキーフレームグループIDを参照します。サブスクライバーは利用可能な最大のキーフレームグループに参加し、その後次のキーフレームまで`motion.delta`データグラムを受信します。

## レイテンシーの調査結果

この評価によって、プロダクションコードはMoQに切り替えられていません。Minamoの測定されたベースラインは、`relay-rs`のWebTransportデータグラムリレーのままです。リリーススモークは、毎回ネイティブなパブリッシュ/サブスクライブデータグラムエコーテストを実行します。MoQに関しては、現在の状態はプロトコルマッピングとリスク分析であり、CI安定レイテンシーベンチマークではありません。

予想されるレイテンシーの影響:

| パス | 予想される動作 | リスク |
|---|---|---|
| 現在の `relay-rs` データグラム | 一つのルームブロードキャストホップ、最新のみのサブスクライバードレイン | 知られたローカルテストパス |
| MoQ 直接リレー | 類似のQUIC/WebTransport基盤に加え、MoQTオブジェクトヘッダー | オブジェクトごとの小さなオーバーヘッド |
| MoQ CDN/リレーメッシュ | スケーラブルなファンアウトとキャッシングセマンティクス | 追加のリレースケジューリング、相互運用性の成熟度は不明 |

MoQTドラフトはオブジェクトデータグラムを定義し、オーバーサイズのデータグラムオブジェクトは小さいデータグラム制限のあるパスでドロップされる可能性があることに注意しています。これはKGM2デルタセマンティクスと一致しますが、製品使用前にMTU検証が必要です。

## ゴー/ノーゴー

決定: 今日、このリポジトリでのプロダクションにはノーゴー。

理由:

- Minamoはすでに低レイテンシールーム用のテスト済みWebTransportリレーパスを持っています。
- MoQTは2026年にまだアクティブなインターネットドラフトであり、ワイヤーの詳細が変更される可能性があります。
- 公共のMoQリレー相互運用ターゲットは、リリーススモークの一部ではありません。
- KGM2は、別の配信レイヤーを追加する前に、製品輸送の展開が必要です。

再訪するタイミング:

- 公共のMoQリレーがCIまたは決定論的なナイトリーロボットで実行できるとき
- Minamoがそのリレーを通じて`motion.delta`および`motion.keyframe`トラックを公開できるとき、p95の一ホップレイテンシーが`relay-rs`に匹敵する場合
- E2EEメタデータ要件がMoQリレーキャッシングメタデータと調整されるとき

それまでの間、大規模なファンアウト研究のための配信オプションとしてMoQを保持し、Minamoの直接WebTransport/WebSocketリレーでプロダクションルームを維持してください。
